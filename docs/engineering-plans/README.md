# Engineering plans

Per-feature spec stubs created by Claude Code as the
roadmap fills in. Each spec is a markdown file with:

- **Goal** (1–3 sentences)
- **Approach**
- **Open questions**
- **Acceptance criteria**

Specs are referenced by `metrics/work-tracker.json`
stories so the user can click from the Kanban view in
the v9 UI directly into the implementation plan.
